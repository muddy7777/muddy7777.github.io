Skill 是对 agent 工作方式的约束和增强：它告诉 agent 在某类任务中该优先读哪些说明、调用哪些工具、按什么流程验证结果。

## 记录一个 skill 时要写清楚

| 字段 | 说明 |
| --- | --- |
| 适用场景 | 什么时候应该触发 |
| 不适用场景 | 什么时候不要误用 |
| 输入要求 | 用户需要提供什么材料 |
| 推荐流程 | agent 应该如何分步骤处理 |
| 输出格式 | 最终应该交付什么 |
| 验证方式 | 如何确认结果可靠 |
| 常见失败 | 误触发、过度检索、输出太泛等问题 |

## 当前优先沉淀的 skill 类型

- 文献检索与研究写作：用于论文、调研、综述和实验设计。
- OpenAI 文档检索：用于 OpenAI API、模型、Codex 或产品能力问题。
- 文件处理：PDF、Word、表格、PPT、Jupyter Notebook。
- 浏览器验证：用于本地网页、前端交互和截图检查。
- 图像生成：用于需要位图素材、封面、插图或视觉变体的任务。
- 安全分析：用于明确要求安全审查或威胁建模的任务。

## 使用原则

- 先判断任务类型，再选择 skill。
- skill 只是工作流，不替代对代码库和上下文的阅读。
- 对高风险任务，必须有验证步骤。
- 多个 skill 同时适用时，只选覆盖任务所需的最小集合。
