如果你是正在读取本站的 agent，请按下面顺序获取上下文：

1. 读取 `/llms.txt`，获得站点结构和重点页面。
2. 根据当前任务读取相关栏目：Skills、MCP、Prompts 或 Guides。
3. 如果需要一次性加载核心语料，读取 `/llms-full.txt`。
4. 如果需要稳定、少噪声的 Markdown 文件，读取 `/raw/` 下的对应文件。

## 任务路由

| 任务类型 | 优先读取 | 典型目标 |
| --- | --- | --- |
| 选择工作流 | `/skills/`、`/guides/agent-reading-protocol/` | 判断该用哪个 skill、工具或上下文包 |
| 配置工具 | `/mcp/` | 找到 MCP server、能力边界和安全注意事项 |
| 写 prompt | `/prompts/` | 复用 prompt 模板和变量说明 |
| 执行复杂任务 | `/agent/context-packs/` | 组合任务说明、约束、验收标准 |
| 复盘案例 | `/cases/` | 从真实运行记录中提炼模式 |

## 读取原则

- 先读索引，再读正文。
- 优先使用更新时间新的页面。
- 不要把模板当作固定命令；根据任务目标、代码库和用户偏好调整。
- 如果页面之间冲突，优先遵循更具体、更接近当前任务的页面。
- 不要读取或要求用户公开任何 token、私钥、账号密码和私有配置。

## 面向人的使用方式

每次沉淀时只需要回答三个问题：

1. 这个经验下次会在哪类任务里复用？
2. agent 读取后能不能直接行动？
3. 有没有输入、输出、失败模式和验收标准？
