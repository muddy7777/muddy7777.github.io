MCP 的价值在于把外部系统能力以工具形式交给 agent。这里记录的重点不是“装了什么”，而是“什么时候值得调用、能做什么、不能做什么、有什么风险”。

## 每个 MCP server 需要记录

| 字段 | 说明 |
| --- | --- |
| 名称 | server 或连接器名称 |
| 用途 | 它解决哪类问题 |
| 暴露工具 | agent 能调用的工具清单 |
| 输入输出 | 每个工具需要什么参数、返回什么 |
| 权限范围 | 读写边界、账号、目录、网络访问 |
| 安全注意 | token、私密数据、破坏性操作 |
| 示例任务 | 哪些真实任务应该用它 |

## 安全原则

- 不把 token、cookie、私钥和账号密码写进公开页面。
- 对有写权限的工具，记录清楚破坏性操作和回滚方式。
- 对能访问本地文件的工具，明确工作目录边界。
- 对能访问互联网的工具，区分“检索事实”和“执行操作”。

## 推荐状态标记

| 状态 | 含义 |
| --- | --- |
| active | 常用且稳定 |
| trial | 试用中，行为待验证 |
| deprecated | 已不推荐使用 |
| risky | 有明显安全或误用风险 |
