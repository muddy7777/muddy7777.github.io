---
title: 代码审查 Prompt
description: 让 agent 以代码审查姿态检查 bug、风险、回归和测试缺口。
---

```text
请对以下改动做代码审查。

审查重点：
- 正确性 bug
- 行为回归
- 数据一致性
- 边界条件
- 并发、性能或资源泄漏风险
- 安全风险
- 缺失测试

要求：
1. 先列发现的问题，按严重程度排序。
2. 每个问题给出文件和行号。
3. 不要把风格偏好当作 bug。
4. 如果没有发现问题，明确说没有发现阻塞问题，并说明剩余测试缺口。

上下文：
{PR 链接、diff、分支、相关文件或任务说明}
```

## 输出格式

```text
发现：
- [P1/P2/P3] 标题
  文件行号：
  问题：
  影响：
  建议：

开放问题：

测试缺口：
```

## 适用场景

- Pull Request 审查。
- 合并前风险检查。
- 让 agent 复查刚完成的代码改动。
